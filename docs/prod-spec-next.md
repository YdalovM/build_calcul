# Build Calcul: спецификация следующего этапа

Документ фиксирует, что осталось реализовать после текущей прод-подготовки.

## 1) Контракт релиза (обязательный минимум)

1. Подключить домен и задать `NEXT_PUBLIC_SITE_URL` в production.
2. Включить веб-аналитику (минимум: события открытия страницы, запуск расчета, нажатие на связанный модуль).
3. Проверить индексацию в Search Console: `sitemap.xml`, `robots.txt`, статус canonical.
4. Подготовить og-image для шаринга и добавить его в metadata.

## 2) Калькуляторы: функциональное усиление

1. Добавить sharable URL для каждого калькулятора (query-параметры).
2. Восстанавливать состояние формы из URL и/или localStorage.
3. Добавить экспорт результата в текстовый формат (копировать расчет + входные параметры).
4. Добавить блок "Что проверить перед закупкой" после каждого расчета.

## 3) SEO-контент: рост под long-tail

1. Для каждого модуля добавить блок типовых ошибок пользователя.
2. Добавить в каждую страницу section с 3-5 узкими интентами "как посчитать ... для ...".
3. Запустить контентные страницы-кластеры под категории:
   - стены/штукатурка;
   - пол/стяжка;
   - плитка;
   - смета/бюджет.
4. Перелинковать кластеры с калькуляторами по принципу "вопрос -> расчет -> следующий шаг".

## 4) Технический SEO и производительность

1. Добавить автоматическую проверку мета-тегов и schema в CI (smoke test).
2. Добавить проверку отсутствия дублированных `title`/`description`.
3. Проверить Core Web Vitals на проде (LCP/INP/CLS) и зафиксировать базовую метрику.
4. Настроить регулярный re-crawl критичных страниц после контентных обновлений.

## 5) UX и доверие

1. Добавить страницу "Методика расчетов" с прозрачным описанием допущений.
2. Добавить единый блок "Ограничения модели" с одинаковой структурой на всех калькуляторах.
3. Добавить легкую форму обратной связи по неточным расчетам (без навязчивых popup).
4. Добавить понятный блок "Что дальше" (маршрут пользователя после расчета).

## 6) Монетизация (после стабилизации трафика)

1. Вводить рекламные блоки только внизу результата, без перекрытия формы.
2. Не использовать полноэкранные interstitial и авто-видео.
3. Отдельно замерять влияние рекламы на глубину и повторные визиты.

## 7) Критерии готовности этапа

Этап считается завершенным, когда:

1. Все калькуляторы поддерживают sharable URL.
2. На каждой странице есть "ошибки/ограничения/следующий шаг".
3. Search Console не показывает критичных ошибок индексирования.
4. Есть базовые метрики поведения и технического качества за последние 14 дней.
